Triage

TREE-ahzh · UK /ˈtriː.ɑːʒ/ · US /triˈɑːʒ/

不是把所有問題一起處理，而是先快速判斷誰最急、誰最重要、誰可以稍後。

今日定位

把 triage 記成「sort under pressure」：資源有限時，不是平均分配，而是先分級。

Core Idea

triage 表示在時間、注意力或資源不足時，先快速分類與排序，決定先處理什麼、延後什麼、轉交什麼。

概念焦點

核心不是解決，而是先做高壓下的優先級判斷。

語氣質地

原本偏醫療，現在在 incident response、bug handling、support queue 與 operations 裡很常見。

使用提醒

不要把 triage 當成 solve；triage 是先分流與判級，不代表問題已經被修好。

Meaning

簡短定義

Triage

表示在事情太多、資源太少時，先快速分類與判急迫性，決定誰要先處理、誰可延後。

不是 Prioritize

prioritize 可以很從容地排順序；triage 帶有資源不足、壓力高、需要快速分級的情境。

Origin

字源來源

triage 經由法文進入英文，核心是 sorting。它原本在醫療與戰地情境裡很重要，因為不可能同時處理所有傷患，必須先判定優先順序。

這個字的重量在於：不是因為你不想全部救，而是因為現實逼你先判斷哪個最不能等。

Memory Hook

記憶鉤子

triage 是 inbox 爆掉時，先救會讓系統失血的那幾件事。

想像急診室同時來很多人，醫護不會先按排隊順序，而是先看誰最危急。工程 incident queue 其實常是同一種思維。

Usage

生活與專業情境

日常版

旅行前待辦很多時，你可能需要先 triage what must happen today and what can wait.

數位 / 專業版

當 crash report 一次湧進來時，團隊第一步往往不是修，而是先 triage severity、scope 與 reproducibility。

延伸版

醫療上的 triage 重點是傷勢分級；工程上的 triage 則常是 incident、bug、ticket 或 alert 的分級處理。

Collocations

常見搭配與語域

Triage 常和 bug、incident、queue、ticket 搭配，因為它幾乎總在高壓輸入過量的場景裡出現。

triage incoming bugs 工程 / QA

表示先把 bug 依嚴重性、影響範圍與可重現性分級。

triage the queue support / operations

表示快速檢視大量待辦，決定哪些要先進場。

triage an incident SRE / incident response

表示先釐清事件級別、使用者影響與應變優先序。

Neighbors

對立與鄰近概念

概念 核心精神 最自然的使用時機
Triage 在壓力與資源不足下，先分級再處理 談 bugs、incidents、tickets、patients 或 overloaded queues 時
Prioritize 一般性的先後排序 當你只是安排順序，不特別強調壓力與資源不足時

Modern Use

現代延伸

現代英文裡，triage 已經穩定進入 software、support、security 與 operations。它幾乎是高壓情境下『先分流再處理』的標準字。

在工程語境中，這個字比 prioritize 更有現場感，因為它包含信息不完整、量太大、不能全部同時處理的現實約束。
