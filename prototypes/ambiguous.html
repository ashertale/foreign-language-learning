Word 19 / adjective

Ambiguous

am-BIG-yoo-us · UK /æmˈbɪɡ.ju.əs/ · US /æmˈbɪɡ.ju.əs/

不是「不清楚」這麼簡單，而是同一句話、訊號或需求可以合理走向兩種以上解讀。

今日定位

把 ambiguous 記成「pulled in more than one direction」：問題不是看不見，而是看見後仍可能分岔。

Core Idea

ambiguous 描述某件事可被兩種以上合理方式解讀。

概念焦點

意義、責任、條件或訊號沒有被釘牢，導致多種解讀同時存在。

語氣質地

正式到日常都能用；在需求、規格、法律、設計、語言學與 debug 報告中特別有用。

使用提醒

不要把 ambiguous 和 vague 混在一起；vague 是輪廓模糊，ambiguous 是可能有多個明確方向。

Meaning

簡短定義

Ambiguous

因為意義、範圍或意圖不夠固定，所以可被理解成一種以上意思。

不是 Vague

vague 是不夠具體；ambiguous 是有兩種以上可能解釋。

一句話或一個訊號
→
多種解讀分岔
→
釐清意圖與條件

Origin

字源來源

ambiguous 來自 Latin 元素，帶有「在周圍、往多個方向移動」的意思。字源畫面像意義被推向不只一條路。

你可以把它想成一個路牌站在岔路口：它不是完全空白，而是讓兩條路都看起來合理。

Memory Hook

記憶鉤子

ambiguous 是偽裝成句子的岔路口。

一句規格如果可以被 firmware、QA、PM 各自解讀成不同實作，它就不是單純寫得短，而是留下了分岔點。

Usage

生活與專業情境

日常版

訊息如果讓人分不清會議是取消還是改時間，就可以說 the message was ambiguous。

數位 / 專業版

一條 ambiguous requirement 可能在 review 時安靜通過，之後變成昂貴的 bug。

延伸版

在 parsing 裡，ambiguous grammar 允許同一個 input 產生一種以上有效結構。

Collocations

常見搭配與語域

Ambiguous 常搭配 statement、requirement、signal、result，指解讀上的分岔。

ambiguous requirement 工程 / 產品

指規格可以被不同人合理解讀成不同實作。

ambiguous signal 系統 / 診斷

指觀察到的訊號不足以唯一指向某個原因。

ambiguous wording 寫作 / 法律

指文字本身留下多重解釋空間。

Neighbors

對立與鄰近概念

概念 核心精神 最自然的使用時機
Ambiguous 同一材料可導向多個合理解讀 句子、規格、訊號或責任界線會分岔時
Vague 輪廓不清、缺少具體資訊 當問題是太籠統，而不是可被清楚地往兩邊解讀

Modern Use

現代延伸

現代英文常用 ambiguous 診斷語言、需求、資料、法律、圖片或行為中可支撐多種解讀的地方。

在工程語境中，說某件事 ambiguous 不只是批評，而是在要求補上條件、threshold、owner 或例子。
